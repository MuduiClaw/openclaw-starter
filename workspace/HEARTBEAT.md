# HEARTBEAT.md — 主 Session 心跳协议

> 心跳触发时执行。职责：监控 cron fleet 健康，不执行具体任务。

## OODA 循环

1. **感知**：读最近消息（用户意图？系统事件？）+ `cron list`（失败？`consecutiveErrors > 0`？）
2. **判断**：用户关注点 + fleet 产出 = 匹配？Job 挂了 = 告警？全绿且有实质产出 = 忽略。
3. **决策**：只在**高价值**或**关键告警**时开口。"一切正常"= 噪音，不说。
4. **行动**：默认回复 `HEARTBEAT_OK`。能自修先自修（timeout ×1.5 上限 1800s；重启 disabled job）。修不了才报告。

## 规则
- **不做具体任务**：不跑扫描、不 fetch，那是 cron 的活。
- **不打招呼**。
- **用户活跃**（最后消息 < 2 分钟）：跳过心跳，直接 `HEARTBEAT_OK`。
- **防重复修复**：改 timeout 前先查最近 1 小时是否已修过。
