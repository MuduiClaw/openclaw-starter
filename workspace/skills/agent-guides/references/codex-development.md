# Codex 代码开发规范

## 核心原则

**所有代码任务必须使用 Codex CLI 完成**（通过 MCP Server 调用），并且每次执行完代码任务后必须继续使用 Codex 完成验证闭环。

**工具链**：

- **Codex CLI**：代码生成与执行
- **Context7**：第三方库文档查询（开发时自动调用）

---

## 验证闭环（5 步强制流程）

```
┌─────────────────────────────────────────────────────────┐
│  1. 产出物完整性确认                                      │
│     └─ 代码文件已生成、依赖已声明、配置已就绪              │
├─────────────────────────────────────────────────────────┤
│  2. 核心功能自测                                          │
│     └─ 编写并运行单元测试 / 集成测试                      │
├─────────────────────────────────────────────────────────┤
│  3. 失败修复循环                                          │
│     └─ 分析错误原因 → 修复 → 重测 → 直到通过              │
├─────────────────────────────────────────────────────────┤
│  4. 用户行为模拟                                          │
│     └─ 模拟真实用户操作路径，验证端到端流程               │
├─────────────────────────────────────────────────────────┤
│  5. 最终确认（全部满足方可交付）                          │
│     ✓ 编译无报错                                         │
│     ✓ 测试全部通过                                       │
│     ✓ 无运行时警告                                       │
│     ✓ 用户真实操作正常                                   │
└─────────────────────────────────────────────────────────┘
```

---

## Codex 调用模式

> Codex 是 CLI 工具，通过 `mcp_codex_codex` MCP 接口调用。开发过程中可配合 `context7` 查询第三方库文档。

### 开发阶段

```
mcp_codex_codex(
  prompt="<具体开发任务描述>",
  workingDirectory="/path/to/project",
  fullAuto=true,  // 启用沙箱自动执行
  model="gpt-5.2-codex"  // 默认模型
)
```

### 验证阶段

```
mcp_codex_codex(
  prompt="验证上一步的代码实现：1) 运行测试 2) 检查编译 3) 模拟用户操作",
  sessionId="<上一步返回的 sessionId>",  // 延续会话上下文
  workingDirectory="/path/to/project"
)
```

> [!NOTE]
> `sessionId` 由 Codex 首次调用成功后自动返回，后续调用传入可延续上下文。若首次调用失败则无 sessionId 返回。

---

## 交付标准

| 检查项 | 通过标准 |
|--------|----------|
| 编译状态 | `exit code 0`，无 error |
| 测试覆盖 | 核心逻辑有测试用例 |
| 警告处理 | 无 warning 或已标注豁免理由 |
| 用户路径 | 主流程端到端可用 |

---

## 失败处理

若验证未通过，**禁止直接交付**。必须：

1. 分析具体错误日志
2. 定位根因（代码逻辑 / 环境配置 / 依赖版本）
3. 修复后重新进入验证闭环
4. 记录修复过程（可选：写入 walkthrough.md）
