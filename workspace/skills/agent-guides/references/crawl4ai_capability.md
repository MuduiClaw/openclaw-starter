# Crawl4AI 使用规范

## 触发条件

当用户要求以下操作时，考虑使用 `crawl4ai` skill：

- "读取这个链接" 或 "总结这篇文章"
- "抓取这个文档"
- "获取网站内容"
- "分析这篇文章"
- "获取页面信息"（无需视觉交互）

## 决策逻辑：Crawl4AI vs Browser Subagent

| 特性 | Crawl4AI | Browser Subagent |
| :--- | :--- | :--- |
| **速度** | ⚡️ 即时（无头模式） | 🐢 较慢（完整渲染） |
| **交互能力** | ❌ 无（只读） | ✅ 完整（点击/输入） |
| **登录认证** | ❌ 困难（除非静态 Token） | ✅ 简单 |
| **JS 渲染** | ✅ 良好（通过 Playwright） | ✅ 完美 |
| **输出格式** | 📄 干净的 Markdown | 🖼️ 截图/DOM |

**经验法则**：

- 如果只需要从公开 URL 获取**文本/知识** → 使用 **Crawl4AI**
- 如果需要**执行操作**（登录、购买、配置）或查看**页面布局** → 使用 **Browser Subagent**
