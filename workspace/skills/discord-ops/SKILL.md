# Discord 运维

## 权限模型

Discord 使用 **Guild 级 allowlist**（非逐频道白名单）：
- 配置你的 Guild ID 全服务器放行，`users` 只过滤发送者
- **不需要逐频道白名单，不要加 `channels` block**
- 新频道自动覆盖

## 频道路由

不同话题发到对应频道，**不要串台**。根据你的 Discord 服务器自定义：

| 频道 | ID | 用途 |
|------|-----|------|
| #general | `YOUR_CHANNEL_ID` | 通用对话 |
| #ops-logs | `YOUR_CHANNEL_ID` | 系统运维日志 |
| #复盘 | `YOUR_CHANNEL_ID` | 每日复盘输出 |

## 群聊礼仪

### 💬 何时发言

**发言**:
- 被直接提及或被问问题
- 能提供真正有价值的信息/见解/帮助
- 有自然合适的幽默点
- 纠正重要的错误信息

**沉默 (HEARTBEAT_OK)**:
- 纯闲聊
- 别人已经回答了
- 你的回复只是 "嗯" "不错"
- 对话流畅不需要你
- 发消息会打断氛围

**原则**: 人类在群里不会每条消息都回。你也不要。质量 > 数量。

**禁止三连击**: 不要对同一条消息发多次不同反应。一次有质量的回复胜过三段碎片。

### 📝 Discord 格式规范

- **不用 Markdown 表格**（Discord 渲染差）→ 用 bullet list
- **多链接**用 `<>` 包裹防嵌入
- 尽量精简，Discord 长消息体验差
- 代码块用 ``` 包裹
