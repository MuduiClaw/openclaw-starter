# Spec: v1.3.1 公开发布清理

- **Status**: done
- **Author**: Partner
- **Created**: 2026-03-11
- **Target**: v1.3.1
- **Oracle Review**: v131-spec-review-api (ITERATE → v2) → v131-spec-review-v2 (ITERATE → v3: grep scope fix)

## 目标

让 openclaw-starter 成为真正可用的公开项目——新用户 clone 后跑 setup.sh，不会遇到私有引用、缺失文件、过期组件。

## 背景

v1.3.0 完成了核心功能（setup.sh 健壮性、Node v25、git gates），但 E2E 验证暴露了以下问题：
- 脚本里残留私有名字（Mudui）和绝对路径
- workspace 缺少 4 个关键模板文件
- Dashboard standalone 过期（v1.0.0，缺 visibleModules/force-dynamic/native addon 修复）
- CHANGELOG 落后于实际提交
- 缺少 LICENSE 文件

## 交付物

### T1: 脱敏 — 全量清理
- **改什么**: 
  - `guardian_agent.py` 中 10 处 `Mudui` → `owner`/`用户`
  - `webapp-testing/SKILL.md` 中 `木对` 引用删除
  - 全项目扫描 `/Users/mudui` `/Users/wangshufu` 等绝对路径残留
- **怎么验**: 
  - `grep -ri "Mudui\|wangshufu\|hxrbot\|木对" . --include="*.md" --include="*.sh" --include="*.py" --include="*.mjs" --include="*.json" --include="*.ts" | grep -v .git` = 0
  - `grep -r "/Users/" . --include="*.md" --include="*.sh" --include="*.py" --include="*.mjs" --include="*.json" --include="*.ts" | grep -v .git | grep -v "example\|template\|placeholder"` = 0
- **不做什么**: 不改 AGENTS.md/MEMORY.md（用户自填模板）
- **影响**: 仅文本替换，不改逻辑

### T2: 添加 workspace 模板文件
- **改什么**: 创建 `workspace/SOUL.md`, `workspace/IDENTITY.md`, `workspace/USER.md`, `workspace/TOOLS.md` 通用模板
- **怎么验**: `ls workspace/{SOUL,IDENTITY,USER,TOOLS}.md` 全部存在；内容是通用骨架 + 注释引导，不含私有信息
- **不做什么**: 不复制 Mac mini 的个人内容
- **影响**: 新增 4 文件

### T3: 打新 Dashboard standalone release
- **改什么**: 
  1. Mac mini 上 rebuild infra-dashboard standalone tarball，包含 visibleModules 过滤 + force-dynamic + native addon(Node v25 ABI 141)
  2. 上传到 GitHub Release `dashboard-v1.1.0`
  3. setup.sh 中 `DASHBOARD_RELEASE_URL` 如果是 `/latest/download/` 格式，无需改动（自动指向最新）；验证 latest release 确实更新
- **怎么验**: 
  1. tarball 内有 `server.js` + `better_sqlite3.node` (ABI 141)
  2. GitHub Release latest 指向 `dashboard-v1.1.0`
  3. `curl -sI $DASHBOARD_RELEASE_URL | grep location` 指向新 tarball
- **不做什么**: 不改 dashboard 代码
- **影响**: GitHub Release 新增 asset
- **风险**: release-dashboard.sh 可能需要调试；native addon 需确认 Node v25 编译

### T4: LaunchAgent 文档化
- **改什么**: README 补充 LaunchAgent 段落说明 plist 生成逻辑 + PATH 注入 + 常见问题
- **怎么验**: README 搜索 "LaunchAgent" 有说明段落
- **不做什么**: 不抽取 plist 模板文件
- **影响**: 纯文档

### T5: 补全 CHANGELOG + 添加 LICENSE
- **改什么**: 
  - CHANGELOG 补入 v1.3.0 后所有 commits + 本次 T1-T4 变更
  - 添加 LICENSE 文件（MIT）
- **怎么验**: 
  - `git log 85c7fc2..HEAD --oneline` 每个 commit 在 CHANGELOG 有对应条目
  - `cat LICENSE | head -1` 包含 "MIT"
- **影响**: 纯文档

### T6: Bump VERSION
- **改什么**: `VERSION` 1.3.0 → 1.3.1
- **怎么验**: `cat VERSION` = `1.3.1`
- **影响**: 版本号

## 不做什么

- 不改 setup.sh 核心逻辑（dashboard URL 已用 `/latest/download/` 自动指向最新）
- 不改 config template
- 不改 cron prompts
- 不新增 skills
- 不碰 Mac mini 的 clawd workspace

## 执行顺序

T1 → T2 → T3 (Dashboard rebuild + upload) → T4 → T5 (CHANGELOG + LICENSE) → T6 → commit → push

## 风险

- T3 (Dashboard rebuild) 如果 `release-dashboard.sh` 有问题需要调试，可能耗时较长
- T2 模板文件内容需要 review，确保对新用户有引导价值而不是空壳
- Native addon (better-sqlite3) 绑定 Node v25 ABI 141，setup.sh 已强制 Node v25 且有 rebuild fallback
- 绝对路径可能藏在非显而易见的位置（.mjs/.json），T1 grep 覆盖范围要广
